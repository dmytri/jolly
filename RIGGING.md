# Rigging

Project tooling values for Shipshape roles. Values only, not procedure.
Procedure lives in the skills. Every role reads this on open.

## Commands

- discover: `npx cucumber-js -p all --dry-run --format message --tags "not @captain and not @shipwright" | node -e 'let s="";process.stdin.on("data",d=>s+=d).on("end",()=>{const u=s.split("\n").filter(Boolean).map(l=>JSON.parse(l)).filter(m=>m.testStepFinished&&m.testStepFinished.testStepResult.status==="UNDEFINED");if(u.length){console.error("undefined steps: "+u.length);process.exit(1);}});'` — executes nothing (dry-run) and exits non-zero when any step is undefined. The tag-free `all` profile in `cucumber.js` carries no tags, so one invocation enumerates every tier and a tier added later is covered with no further wiring.
- focused: `bash -c 'f="$1"; shift; t=$(mktemp); npx cucumber-js -p all "${f%%:*}" --name "^${f#*:}$" --tags "not @captain and not @shipwright" "$@" 2>&1 | tee "$t"; s=${PIPESTATUS[0]}; grep -q "^0 scenarios" "$t" && { echo "focused: selected no scenario for $f" >&2; rm -f "$t"; exit 1; }; rm -f "$t"; exit $s' _ "{scenario}"`. Runs on the tag-free `all` profile, so a target in ANY configured tier is selectable. Cucumber ANDs a profile's tags with the CLI's, so the default profile's `tags: "not @eval"` silently selected NOTHING for an `@eval` target and exited 0; `all` carries no tags, so a tier added later is covered with no further wiring. Selecting no scenario exits 1, naming the reference, so a mistyped or mis-tiered target reddens instead of reading as a pass.
- broad: `npx cucumber-js -p logic --format message:coverage/weather/logic.ndjson --tags "@logic and not @captain and not @shipwright"`
- broad-sandbox: `npx cucumber-js -p sandbox --format message:coverage/weather/sandbox.ndjson --tags "@sandbox and not @captain and not @shipwright"`
- broad-sandbox-serial: `npx cucumber-js -p sandboxSerial --format message:coverage/weather/sandboxSerial.ndjson --tags "@sandbox and not @captain and not @shipwright"`
- broad-eval: `npx cucumber-js -p eval --format message:coverage/weather/eval.ndjson --tags "@eval and not @captain and not @shipwright"`
- coverage: `npx c8 --reporter=text --reporter=json -- npx cucumber-js -p logic --format message:coverage/weather/instrumented/logic.ndjson --tags "@logic and not @captain and not @shipwright"`
- coverage-sandbox: `NODE_OPTIONS=--max-old-space-size=4096 npx c8 --clean=false --reporter=text --reporter=json -- npx cucumber-js -p sandbox --format message:coverage/weather/instrumented/sandbox.ndjson --tags "@sandbox and not @captain and not @shipwright"`
- coverage-sandbox-serial: `NODE_OPTIONS=--max-old-space-size=4096 npx c8 --clean=false --reporter=text --reporter=json -- npx cucumber-js -p sandboxSerial --format message:coverage/weather/instrumented/sandboxSerial.ndjson --tags "@sandbox and not @captain and not @shipwright"`
- coverage-eval: `NODE_OPTIONS=--max-old-space-size=4096 npx c8 --clean=false --reporter=text --reporter=json -- npx cucumber-js -p eval --format message:coverage/weather/instrumented/eval.ndjson --tags "@eval and not @captain and not @shipwright"`
- step-usage: `npx cucumber-js -p all --dry-run --format usage-json --tags "not @captain and not @shipwright"` — emits one `usage-json` array covering every configured tier. The tag-free `all` profile enumerates every scenario in one invocation, so no tier is missed and no step definition reads as a false-positive orphan. Measured at harbour 2026-07-20: 1145 step definitions, 21 with zero usage.
- reclaim: `npm run reclaim` — standalone preflight that deletes stale `jolly-cannon-fodder`-namespaced leftovers (Cloud environments + local scratch dirs) without running any tier; the same reclamation also runs automatically at the start of every cucumber invocation (BeforeAll, `features/support/hooks.ts`)
- plank-inventory: `rg -n '@planks' src bin` — lists `@planks` and `@planks-provisional` alike across the implementation paths. Recursive `grep` is denied to internal roles by the Shipshape Bash custody hook, because it never reads the ignore artifact that hides the Captain-only notes; `rg` honours it, so this is the runnable form. Presence only: plank FORM and FRESHNESS are observed against the TypeScript AST by the `@logic @invariant` plank checks in feature `methodology-conformance` (`features/support/plank-conformance.ts`), never by this text search.
- typecheck: `npm run typecheck`
- lint: `npx gplint "features/*.feature"`
- conformance: `npx cucumber-js --profile logic --tags "@logic and (@property or @invariant) and not @captain and not @shipwright"` — runs the project's whole conformance family: the structural `@property` scenarios (module-layering boundaries, single env-creation seam, single command-surface parser seam for the global output flags, live-by-design) discharged by the ts-morph conformance checker, AND the `@invariant` methodology and verification-economy scenarios (perturbation quiescence, watchbill shape, plank form, plank-pattern join, spec comments, credentialed-tier failure, wake record, prompt-observed waits and reads). Both tags are this project's dialect of `@conformance`: a scenario attesting the project's own method rather than the product. Selecting `@property` alone silently skipped the two REQUIRED methodology checks, so a role running this command to discharge methodology conformance got a green over 31 scenarios that never touched it. Measured at refit: `@property` 31, `@invariant` 17, family 48.

## Tiers

- default: @logic. Fast behaviour tier, run in parallel. Exercises real behaviour against the `.env` test env per the live-by-design policy in `AGENTS.md`. Credentials are present by fitting-out; verification reads them from the environment and runs every target. A target whose credential or capability is absent fails as a fitting-out blocker, naming what fitting-out must provide.
- sandbox: @sandbox. Requires `JOLLY_SALEOR_CLOUD_TOKEN` and a Vercel CLI session, both present by fitting-out and read from the environment; verification runs every target and never gates on credential presence. A target whose credential is absent fails as a fitting-out blocker. The harness provisions `jolly-cannon-fodder`-namespaced Saleor Cloud and Vercel resources. Most scenarios share ONE store, deliberately cached across cucumber invocations via a persistent marker file (created once, reused while healthy, self-heals under a freshly-named replacement if unreachable — never torn down); only scenarios that test store/environment creation itself (`@creates-env`) provision their own disposable one and tear it down. The licensed full-pipeline proof (`@pipeline`) is ONE scenario — the end-to-end operational-readiness proof — and it is the only scenario entitled to the whole toolchain chain, which runs at most once per run as the shared pipeline; every other scenario, the interactive completion included, asserts against the shared pipeline's satisfied state, and the spend-ledger checks in feature `verification-economy` enforce the licence. `@pipeline` and `@creates-env` run serial, and the serial leg runs BEFORE the parallel leg: the licensed set is the primary seam proofs, so a broken creation or chain seam reds first, before the derivative satisfied-state scenarios spend against it. The order is free: the two legs are sequential either way. Stale leftovers from any run are reclaimed proactively at the start of every invocation (`npm run reclaim` / BeforeAll), not lazily on next same-tier run.
- eval: @eval. Required green/red gate driving the live baseline agent. Requires `HARNESS_OPENROUTER_API_KEY` and `HARNESS_EVAL_MODEL`, present by fitting-out. Runs in the full-tier boundary and MUST pass; never skipped. A single live-agent timeout MAY be absorbed by a bounded in-scenario retry, persistent failure reds. This is the ONLY tier that invokes a model; every other tier reports zero model invocations and zero tokens, per feature `verification-economy`. Jolly's expensive service effects are served from golden captures recorded by the licensed `@pipeline` sandbox runs (feature 025), so this tier needs no Vercel session and creates no cloud resource; its cost is the agent's turns.
- budget: 3640 — the full-regression wall-clock ceiling in seconds, the SUM of the tier budgets under the sequential `order` below: logic 650 + sandbox-serial 900 + sandbox 1850 + eval 240. The first full sequential sweep measured 2973.6s on 2026-07-22 (logic 589.5 + sandbox-serial 588.2 + sandbox 1681.7 + eval 114.2), which fits. Supersedes 3165, the same sum before the 2026-07-22 tier re-ratchets, and before that the 2250 laned-WINDOW ceiling retired with `lanes`; a window ceiling judged from overlapping launches cannot describe legs that now run one at a time. Its consumer is feature 030's reclamation age gate, which treats a namespaced leftover as stale once it is older than this value, since no live invocation can outlive the whole regression's ceiling. No scenario judges a laned window against it any longer.
- budget-logic: 650 — measured 589.5s at 220 scenarios on 2026-07-22 in the first sequential sweep, ceiling carries ~10% headroom. Supersedes 375, which was measured at 341s over 224 scenarios before the interactive scenarios reached their current cost. The tier is NOT uniformly slow: median scenario 0.70s against a mean of 5.30s, and TWELVE PTY scenarios cost 569s of the tier's 1165.6s of scenario time, 49%, led by "A failed setup stage closes honestly" at 79.3s and "Progress is shown in place on stderr" at 67.6s. Parallelism is already 1.98x at the declared 2 workers, so no worker change touches this. Per dk ruling 2026-07-22 the 569s is harbour work, and this ceiling re-ratchets DOWN once it lands, never a bare value edit.
- workers-sandbox: 1 — the DECLARED worker count for the @sandbox parallel profile, per dk ruling 2026-07-21, read as configured rather than derived at run time. This VM is resource-limited (7.9 GB) and the heavy @sandbox toolchain scenarios (deploy's real Vercel build, storefront prepare) OOM-kill the run when they co-reside; observed twice on 2026-07-21. The VM's capacity is an operator fact that does not change between runs, so the count is declared here rather than derived from the weather record: a run-time derivation computes this same value and adds its own failure modes to guard. Declared in the rigging rather than pinned in `cucumber.js`, so the value is legible instead of hidden in the runner configuration. Raise it when the tier runs on a box whose memory the co-resident toolchain does not exhaust.
- workers-logic: 2 — the DECLARED worker count for the @logic parallel profile. Logic is light enough never to exhaust this box's memory, and parallelism there earns its keep: 242 scenarios in 317s at 2 workers.
- budget-sandbox: 1850 — the parallel profile's wall at the declared 1 worker, measured 1681.7s on 2026-07-22 over 53 scenarios all green, with ~10% headroom. Supersedes 1650, measured 1479s on 2026-07-21 on a quieter box; the 1.9% overrun against it was contention, not growth. That run recorded peak RSS 2.4 GB against an 8.3 GB ceiling and attributed no out-of-memory kill to itself, so the declared 1 worker holds with room. Re-ratchet from a fresh measurement if the worker count is ever raised.
- budget-sandbox-serial: 900 — the serial leg's wall, measured 588.2s on 2026-07-22 over 3 scenarios all green, superseding an 811s measurement of 2026-07-19. The ceiling is held at 900 rather than ratcheted down to the fresh measurement: this leg runs the licensed @pipeline and @creates-env scenarios, whose cost is dominated by real cloud provisioning and a full toolchain chain, so its wall swings with service latency rather than with this repo's code.
- budget-eval: 240 — measured 114.2s on 2026-07-22 over 4 scenarios all green. Held at 240 rather than ratcheted down: the tier drives a live agent whose turn count varies run to run, and a bounded in-scenario retry may absorb one timeout, so the ceiling carries room for that variance by design.
- weather: coverage/weather/ — the wake's weather record. Every `broad-*` and `coverage-*` command carries `--format message:coverage/weather/<tier>.ndjson`, so a tier run writes its own cucumber message stream, carrying per-test-case nanosecond durations, by construction. Read as the per-scenario duration source for the budget-fit check and for the harbour verification-economy audit. A `<tier>.ndjson` older than the tier's last run is a rigging defect, not a stale file to work around: the tier command owns the write. Instrumented runs write `coverage/weather/instrumented/<tier>.ndjson` instead: the `coverage-*` commands carry c8 overhead and harbour contention, so their records never stand as the operational record the budget-fit check reads.
- runrecord: coverage/weather/runrecord.ndjson — the wake's voyage run record, one JSON object per fresh green run, git-ignored under `coverage/`.
- order: tier-tag watches run one tier at a time — @logic (`broad`), then @sandbox serial leg (`broad-sandbox-serial`), then @sandbox parallel leg (`broad-sandbox`), then @eval (`broad-eval`). Cheapest-first is the default ordering but @eval is DEPENDENT and therefore runs last, not first: its golden captures name a prepared-storefront template under `/tmp` that only the @sandbox tier's licensed shared-pipeline provisioning builds, so @eval run against a cold `/tmp` fails every scenario in ~2s naming `broad-sandbox` as its remedy. Observed 2026-07-22 on the first sequential sweep. Concurrent lanes masked this: the template persisted between runs, so the dependency was never exercised from cold. @logic stays first because it is cheap and independent. Each tier still runs at its own declared internal worker count. Concurrent tier lanes were adopted 2026-07-19 and retired 2026-07-22: on this 7.9 GB box the overlap OOM-killed a sweep and took two sibling lanes down with it, and the run-scoped isolation of feature 030 stays load-bearing regardless, for @logic's own workers and for foreign agents sharing this box. Sequential legs sum to ~41 min against the laned window's 37.5 min that was never achieved.

## Dependencies

- policy: latest-stable. Every dependency tracks the registry's latest stable release, upgraded on sight rather than pinned, and proven by the tier runs. Add a new dependency only when a spec requires it, and select the latest stable version when adding one. A version held BELOW latest stable is a defect unless this file records the reason and the condition that retires it; absent that record, a role upgrades it. npm's engine-aware resolution silently selects an OLDER version when a dependency's `engines` range does not cover this package's declared `engines` floor, so install with an explicit `@latest` and confirm the resolved version rather than trusting a bare `npm install`.
- dependency-audit: `npm outdated` — reports every dependency behind latest stable. No derived check enforces this section against `package.json`, so a dependency recorded here but absent from the manifest, or installed but unrecorded, is invisible to verification. That gap let `c8` ship undeclared and `happy-dom` linger unused; a conformance scenario joining the two is a standing Captain finding.
- yaml: runtime parser for `assets/skills/jolly/recipe.yml`, required by feature `recipe-identifiers-from-asset` (`deriveRecipeIdentifiers`). Version constraint lives in `package.json`.
- @bomb.sh/args: runtime argument parser for the shipped CLI. The single parser call in `src/index.ts` is the one command-surface parser seam the global output flags `--json`/`--quiet`/`--yes` reach every command through, enforced by feature 006's global-output-flags `@property` scenario.
- @bomb.sh/tab: runtime shell-completion library for the shipped CLI, backing `src/lib/completion.ts` and the `jolly completion` command. At 0.0.21, upgraded from 0.0.19 at harbour 2026-07-20 under the `latest-stable` policy.
- @clack/prompts: runtime interactive-prompt library for the shipped CLI, backing the interactive `jolly start` prompts and the stage progress region in feature 027.
- @cucumber/cucumber: the Gherkin runner every tier command invokes, installed with the rigging. At 13.2.0, upgraded from 13.1.1 at harbour 2026-07-20 under the `latest-stable` policy.
- @types/node: dev-only Node type definitions the `typecheck` gate resolves against. Not a runtime dependency of the shipped CLI.
- esbuild: dev-only bundler the `build`, `prepack`, and `prepublishOnly` scripts invoke to produce `dist/index.js` for the npm outbound target. Not a runtime dependency of the shipped CLI.
- @earendil-works/pi-coding-agent: dev-only baseline coding agent for the `@eval` tier, required by feature `025-agent-skill-affordance-eval`. It is SPAWNED as a binary (`node_modules/.bin/pi`), never imported, so the module graph cannot see it and a dead-code analyzer reports it unused. It is not unused. It is also the source of record for verification economy: it writes its own per-turn `usage` (model invocations, prompt and completion tokens) to its session JSONL, read via `--session-dir`, per feature `verification-economy`.
- c8: dev-only V8 coverage tool the `coverage`, `coverage-sandbox`, `coverage-sandbox-serial`, and `coverage-eval` commands invoke, required by the harbour regression per the Shipwright skill. Installed at harbour after the refit found every `coverage*` command invoking `npx c8` with c8 absent from both `package.json` and the lockfile, so each harbour run fetched an unpinned tool from the network. Not a runtime dependency of the shipped CLI. A bare `npm install c8` resolves to 11, not latest: c8 12 requires node `^20.19.0 || ^22.12.0 || >=23` and this package declares `engines: >=20.12.0`, which covers 20.12 to 20.18, so npm's engine-aware resolution picks the older major. Install it with an explicit `@latest`; the dev runtime is node 24, which satisfies c8 12.
- typescript: the `typecheck` gate's compiler, at 7. TypeScript 7 is the native port: its main entry exports only the version, and the compiler API moved to `./unstable/*`, which the TypeScript team labels unstable. Nothing in this project imports the `typescript` compiler API. AST work goes through `ts-morph`, which vendors its own TypeScript through `@ts-morph/common` and is therefore unaffected by the port. Keep it that way: an import of `typescript/unstable/*` would be this tree's only unstable dependency, breakable by any 7.x release.
- ts-morph: dev-only TypeScript-AST library backing the verification layer's structural conformance checker, required by features `module-boundary-conformance`, `single-creation-seam`, and feature 006's global-output-flags `@property` scenario. The checker (verification support) walks the source with ts-morph to enforce the module-layering import boundaries (resolution-accurate, via resolved import source files), the single env-creation seam (the `create store --create-environment` CLI-spawn call pattern that a module-graph tool cannot see), and the single command-surface parser seam (the global output flags `--json`/`--quiet`/`--yes` are declared once in `GLOBAL_BOOLEAN_FLAGS` and reach every command through the one `@bomb.sh/args` parser call in `src/index.ts`). Not a runtime dependency of the shipped CLI.

## Outbound

- outbound: npm
- ship: `npm publish` — the `prepublishOnly` script builds `dist/index.js` first
- verify: `npm view @dk/jolly version` reports the released version and the installed `npx @dk/jolly --help` runs the published bundle
- outbound: vercel-homepage
- ship: `cd assets/homepage && npx vercel deploy --prod --yes` — Vercel project `homepage`, linked via `assets/homepage/.vercel/project.json`
- verify: the deployed `*.vercel.app` homepage serves and its `/setup` rewrite matches the shipped `assets/homepage/setup.md`
- policy: verify the published artifact and the deployed homepage, not only the local tree. After npm publish, verify against the local clean tree while CDN propagation settles (a stale-tarball window is expected and rides through), then verify the published package.
