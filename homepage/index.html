Jolly

Saleor's Hydrogen for the agentic age.

Jolly helps your own AI agent set up a fully operational, end-to-end commerce storefront on Saleor Cloud — and then helps the agent iterate on it with you. Jolly is built on Saleor Cloud and depends on it; Jolly does not replace Saleor.

The primary path starts by copying the instructions below into your agent. Setup takes minutes, with minimal manual steps — only account creation, browser OAuth consent, and pasting secret keys need a human; everything else is automated with safe defaults.

Copy this to your agent to get started

Paste the prompt below into your AI agent. It points the agent at the Jolly setup guide and the Jolly CLI.

You are setting up an end-to-end Saleor Cloud storefront with Jolly.
1. Read the Jolly agent setup guide at https://jolly.example.com/setup-guide and follow it.
2. Run Jolly via npx (no global install): npx @saleor/jolly init
3. Use the Jolly CLI to install and manage the required skills: npx @saleor/jolly skills install
4. Then run the orchestrated setup: npx @saleor/jolly start

The setup guide at the linked URL carries the full context, MCP server details, and workflow guidance for your agent. (The canonical URL is still a placeholder; the committed guide lives next to this page as setup-guide.md.)

How it works — the v1 flow

The v1 journey your agent drives with Jolly:

  1. Copy the prompt above into your agent; the agent reads the setup guide.
  2. The agent initializes Jolly and installs the Saleor agent skills through the Jolly CLI.
  3. The agent checks whether you already have a Saleor store (existing store connection) or want to register a new store on Saleor Cloud.
  4. Saleor Cloud setup or connection: register/connect the store and authenticate.
  5. Configurator-based store configuration: Jolly applies a starter recipe via saleor/configurator so the store is immediately operational.
  6. Storefront creation: Jolly clones Saleor Paper (the official saleor/storefront template) as the baseline.
  7. Stripe test-mode checkout setup: you paste two Stripe test mode keys; Jolly does the rest.
  8. Vercel deployment of the storefront, with required environment variables configured.
  9. Automatic trusted-origin updates: Jolly updates Saleor allowed/trusted origins for the deployed URL where APIs allow.
  10. Final verification: deployed product browsing, cart, and checkout through to the Stripe test payment step.

After setup, the iteration phase begins: your agent and Jolly help you iterate and customize your commerce experience — Jolly contributes diagnostics (jolly doctor), upgrades, and live store access config, while your own agent and workflow do the customization.

Jolly's role

  • Jolly empowers your agent as a capability layer; your agent remains the workflow orchestrator, explainer, and approval manager.
  • Speed promise: minimal human steps — only account creation, OAuth consent, and pasting secret keys. Working storefront in minutes.
  • The Saleor MCP server (mcp.saleor.app) provides read-only live store data access after setup; Jolly handles setup automation and diagnostics.

V1 scope and boundaries

  • V1 supports Saleor Cloud only — no self-hosted Saleor.
  • V1 scope: Saleor Cloud, the Paper storefront, a Configurator starter recipe, Stripe test mode, Vercel.
  • Jolly is not a Saleor backend and does not replace Saleor Dashboard.
  • Post-setup iteration uses your own agent and workflow; Jolly supports it with diagnostics and upgrades.
  • Jolly is not the old (deprecated) Saleor CLI and does not depend on it.
  • No telemetry, no Jolly-owned auth or licensing.
